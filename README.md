# IAP-Bootloader

基于 STM32F4xx 的串口 IAP（In-Application Programming）升级引导程序。支持通过 UART 协议对应用固件进行擦除、烧写、校验，并在启动时自动检测是否需要进入升级模式。

## 功能特性

- 基于 **STM32F40/41xxx**，使用标准外设库（SPL）
- 通过 **USART3**（115200 8-N-1）接收升级指令
- 支持 Flash **擦除 / 烧写 / 校验**操作
- 启动时通过三种方式判断是否进入升级模式（见[启动流程](#启动流程)）
- 应用程序通过 **Magic Header** + **CRC32** 完整性校验
- 集成 **EasyLogger** 日志库，可通过 USART1 输出调试信息
- 支持系统复位指令与直接跳转到应用程序指令

## Flash 内存布局

| 区域            | 起始地址     | 大小   | 说明                        |
|-----------------|--------------|--------|-----------------------------|
| Bootloader      | `0x08000000` | 48 KB  | 本项目可执行代码            |
| Magic Header    | `0x0800C000` | —      | 固件元信息（类型/地址/CRC） |
| Application     | `0x08010000` | ≤464 KB| 用户应用程序                |
| Flash 总容量    | `0x08000000` | 512 KB | STM32F4xx 内部 Flash        |

> **注意**：Bootloader 区域（`0x08000000` ~ `0x0800BFFF`）受写保护，ERASE / PROGRAM 指令不可操作此范围。

## 硬件说明

### UART

| 外设    | 引脚              | 波特率  | 用途             |
|---------|-------------------|---------|------------------|
| USART1  | PA9 (TX), PA10 (RX) | 115200 | 调试日志输出     |
| USART3  | PB10 (TX), PB11 (RX) | 115200 | IAP 升级协议通信 |

### LED

| 名称  | GPIO 引脚 | 说明                 |
|-------|-----------|----------------------|
| LED1  | PE9       | 进入升级模式时点亮   |
| LED2  | PB0       | 预留                 |
| LED3  | PB1       | 预留                 |

### 按键

| 名称  | GPIO 引脚 | 功能                               |
|-------|-----------|------------------------------------|
| KEY1  | PA0       | 预留                               |
| KEY2  | PC4       | 预留                               |
| KEY3  | PC5       | 启动时按住进入升级模式；升级中按下触发系统复位 |

## 通信协议

所有通信通过 **USART3** 进行，上位机发送请求包，Bootloader 回复响应包。

### 数据包格式

#### 请求包（上位机 → Bootloader）

| 字段    | 大小   | 说明                       |
|---------|--------|----------------------------|
| Header  | 1 字节 | 固定为 `0xAA`              |
| Opcode  | 1 字节 | 操作码（见[指令列表](#指令列表)） |
| Length  | 2 字节 | Payload 长度（小端序）     |
| Payload | N 字节 | 指令参数数据               |
| CRC16   | 2 字节 | Header~Payload 的 CRC16 校验（小端序） |

#### 响应包（Bootloader → 上位机）

| 字段    | 大小   | 说明                       |
|---------|--------|----------------------------|
| Header  | 1 字节 | 固定为 `0x55`              |
| Opcode  | 1 字节 | 与请求包 Opcode 相同       |
| Errcode | 1 字节 | 错误码（见[错误码](#错误码)）|
| Length  | 2 字节 | Data 长度（小端序）        |
| Data    | N 字节 | 响应数据（可为空）         |
| CRC16   | 2 字节 | Header~Data 的 CRC16 校验（小端序） |

### 指令列表

| Opcode | 名称    | Payload 格式                                  | 说明                          |
|--------|---------|-----------------------------------------------|-------------------------------|
| `0x01` | INQUERY | `subcode(1B)`: `0x00`=版本, `0x01`=MTU        | 查询 Bootloader 版本或最大传输单元 |
| `0x81` | ERASE   | `addr(4B) + size(4B)`                         | 擦除指定 Flash 区域            |
| `0x82` | PROGRAM | `addr(4B) + size(4B) + data(size B)`          | 向指定地址烧写数据             |
| `0x83` | VERIFY  | `addr(4B) + size(4B) + crc32(4B)`             | 校验指定 Flash 区域的 CRC32    |
| `0x21` | RESET   | 无                                            | 触发系统复位                   |
| `0x22` | BOOT    | 无                                            | 验证并跳转到应用程序           |

> - `addr` / `size` / `crc32` 均为 4 字节无符号整数，小端序。
> - 单次 PROGRAM 最大 Payload 为 **4104 字节**（4096 字节数据 + 8 字节地址/大小字段）。

### 错误码

| 值     | 名称               | 说明               |
|--------|--------------------|--------------------|
| `0x00` | ERR_OK             | 操作成功           |
| `0x01` | ERR_OPCODE         | 未知操作码         |
| `0x02` | ERR_OVERFLOW       | 数据溢出           |
| `0x03` | ERR_TIMEOUT        | 接收超时           |
| `0x04` | ERR_FORMAT         | 数据包格式错误     |
| `0x05` | ERR_VERIFY         | CRC 校验失败       |
| `0xFF` | ERR_PARAM          | 参数错误（地址越界或访问保护区域）|

## 启动流程

上电后，Bootloader 按以下顺序依次检查是否需要进入升级模式：

```
上电
  │
  ▼
1. Magic Header 检查：Magic Header 无效 或 应用程序 CRC32 不匹配
  │  → 是：进入升级模式
  │
  ▼
2. 按键检查（3 秒内持续按住 KEY3）
  │  → 是：进入升级模式
  │
  ▼
3. UART 数据检查（3 秒内 USART3 收到数据）
  │  → 是：进入升级模式
  │
  ▼
以上均否 → 跳转到应用程序（0x08010000）
```

进入升级模式后：
- **LED1** 点亮
- 循环等待并处理来自 USART3 的升级指令
- 升级模式中按下 **KEY3** 可触发系统复位

## Magic Header

Magic Header 存储于 Flash `0x0800C000`，用于描述应用固件的元信息并提供完整性校验。

| 字段          | 偏移   | 大小    | 说明                          |
|---------------|--------|---------|-------------------------------|
| magic         | 0x00   | 4 字节  | 魔数，固定为 `0x4D414749`（`"MAGI"`） |
| bitmask       | 0x04   | 4 字节  | 有效字段位掩码                |
| data_type     | 0x20   | 4 字节  | 固件类型（`0` = APP）         |
| data_offset   | 0x24   | 4 字节  | 固件相对 Magic Header 的偏移  |
| data_address  | 0x28   | 4 字节  | 固件实际加载地址              |
| data_length   | 0x2C   | 4 字节  | 固件数据长度（字节）          |
| data_crc32    | 0x30   | 4 字节  | 固件 CRC32 校验值             |
| version       | —      | 128字节 | 固件版本字符串                |
| this_address  | —      | 4 字节  | Magic Header 自身存储地址     |
| this_crc32    | —      | 4 字节  | Magic Header 结构体自身 CRC32 |

Bootloader 启动时先校验 Magic Header 自身 CRC32，再校验固件 CRC32，两项均通过才认为固件有效。

## 目录结构

```
IAP-Bootloader/
├── app/                    # 应用层代码
│   ├── main.c              # 入口：硬件初始化、日志初始化、进入 bootloader_main
│   ├── bootload.c          # Bootloader 核心逻辑：启动判断、协议处理、Flash 操作
│   ├── board.c / board.h   # 板级硬件描述（LED、按键引脚）
│   ├── magic_header.c / .h # Magic Header 读取与校验
│   ├── jumpapp.s           # 汇编实现：设置栈指针并跳转到应用程序
│   ├── bitops.h            # 位操作工具宏
│   └── utils.h             # 字节序读写工具宏
├── driver/                 # 外设驱动
│   ├── bl_usart.c / .h     # USART3 IAP 通信驱动（中断接收）
│   ├── usart.c / .h        # USART1 调试输出驱动
│   ├── stm32_flash.c / .h  # STM32 内部 Flash 擦除/烧写驱动
│   ├── tim_delay.c / .h    # 基于 TIM6 的毫秒延时
│   ├── led.c / .h          # LED 驱动
│   └── key.c / .h          # 按键驱动
├── firmware/               # STM32 固件库
│   ├── cmsis/              # CMSIS 核心头文件及设备文件
│   └── driver/             # STM32F4xx 标准外设库（SPL）
└── third_lib/              # 第三方库
    ├── easylogger/         # EasyLogger 轻量级日志库
    ├── crc/                # CRC16 / CRC32 实现
    └── ringbuffer/         # 环形缓冲区
```

## 第三方库

| 库名        | 说明                                          |
|-------------|-----------------------------------------------|
| EasyLogger  | 轻量级嵌入式日志库，支持多级别日志和格式配置  |
| CRC16/CRC32 | 用于数据包完整性校验和固件 CRC 校验           |
